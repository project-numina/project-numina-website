# Mémo : Workflow d'édition Project Numina (Decap/Netlify CMS)

## Accès à l'admin
- Tous les éditeurs ayant un compte GitHub avec accès au repo peuvent se connecter à l'admin via /admin/.
- Chacun peut éditer, ajouter ou corriger le contenu selon ses droits GitHub.

## Collections disponibles
- **Communauté** : gestion des membres, avatars, tags, etc.
- **Cartes Projets** : gestion des projets, statuts, images, descriptions, etc.
- **Blog** : gestion des articles d'actualité, annonces, etc.

## Workflow d'édition

### Communauté & Cartes Projets
- Toute modification (ajout, édition, suppression) est enregistrée et publiée directement sur le repo GitHub.
- Les changements sont visibles sur le site dès le prochain déploiement.
- Utiliser ces sections pour tenir à jour l'équipe et les projets.

### Blog
- Un mode "Draft" (brouillon) est disponible pour chaque article.
- Créez ou éditez un article, enregistrez-le en mode "Draft" pour relecture ou validation.
- Quand l'article est prêt, passez-le en "Published" pour publication sur le site.
- Ce workflow permet de préparer des contenus à l'avance et de les publier au bon moment.

## Bonnes pratiques
- Privilégier le mode "Draft" pour les contenus sensibles ou à valider.
- Toute modification via l'admin est versionnée dans GitHub (historique, rollback possible).
- Pour toute question ou besoin de validation, utilisez les statuts ou communiquez avec l'équipe.
